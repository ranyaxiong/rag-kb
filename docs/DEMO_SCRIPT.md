# 任务取消功能演示脚本

## 🎬 演示场景设计

### 场景 1: 前端界面演示（推荐给最终用户）

#### 准备工作
- 启动系统：`make dev` 或 `make docker-run`
- 准备一个大型PDF文件（建议 > 10MB）
- 打开浏览器访问 http://localhost:8501

#### 演示步骤

**第1步：展示上传界面**
```
旁白：这是我们的文档上传界面，支持PDF、Word等多种格式。
操作：指向"📤 上传文档"区域
```

**第2步：上传大型文档**
```
旁白：现在我上传一个20MB的扫描版PDF文件。
操作：
  1. 点击"选择文件"
  2. 选择大型PDF
  3. 点击"📤 上传文件"
```

**第3步：查看处理提示**
```
旁白：上传成功后，系统提示文档正在后台处理。
      注意这里有两个按钮：查看状态和取消处理。
操作：指向两个按钮
显示：
  ✅ large.pdf 上传成功!
  🔄 文档正在后台异步处理中...
  
  📋 大型文档处理提示：
  • 🔍 扫描版PDF需要OCR文字识别，可能需要3-10分钟
  • 🛑 如需取消处理，请点击下方取消按钮
  
  [📊 查看处理状态]  [🛑 取消处理]
```

**第4步：查看处理状态**
```
旁白：我们先查看一下当前的处理进度。
操作：点击"📊 查看处理状态"
显示：
  📊 状态: processing | 进度: 30% | 正在解析文档内容...
```

**第5步：展示正在处理的文档列表**
```
旁白：在上方的"正在处理的文档"列表中，可以看到所有进行中的任务。
操作：展开"🔄 正在处理的文档 (1)"
显示：
  📄 large.pdf  [📊 状态]  [🛑 取消]
```

**第6步：取消任务**
```
旁白：如果我们发现上传错了文件，或者不想等待了，
      可以点击"取消处理"按钮。
操作：点击"🛑 取消处理"
显示：
  ✅ 已取消 large.pdf 的处理
  📝 取消请求已发送，任务将在下一个检查点停止
```

**第7步：验证取消结果**
```
旁白：几秒钟后，任务就会停止。我们再查看一下状态。
操作：点击"📊 查看处理状态"（如果按钮还在）
显示：
  ⚠️ large.pdf 已被取消
```

**第8步：确认文档未添加**
```
旁白：在右侧的文档列表中，我们看不到这个文档，
      说明取消成功，文件已被清理。
操作：指向右侧文档列表
```

### 场景 2: API演示（推荐给开发者）

#### 准备工作
- 启动系统
- 准备测试文件
- 打开终端

#### 演示步骤

**第1步：上传文档**
```bash
# 旁白：首先，我们通过API上传一个文档
curl -X POST "http://localhost:8000/api/documents/upload-async" \
  -F "file=@test.pdf"

# 响应
{
  "success": true,
  "message": "文件上传成功，已加入处理队列",
  "document_id": "abc-123-def",
  "status": "queued"
}

# 旁白：记下这个 document_id，我们后面会用到
```

**第2步：查询任务状态**
```bash
# 旁白：查询任务的当前状态
curl "http://localhost:8000/api/documents/status/abc-123-def"

# 响应
{
  "status": "processing",
  "progress": 45,
  "message": "正在解析文档内容..."
}

# 旁白：任务正在处理中，进度45%
```

**第3步：检查是否可取消**
```bash
# 旁白：我们先检查这个任务是否可以取消
curl "http://localhost:8000/api/documents/cancel-status/abc-123-def"

# 响应
{
  "success": true,
  "document_id": "abc-123-def",
  "status": "processing",
  "cancellable": true,
  "progress": 45,
  "message": "正在解析文档内容..."
}

# 旁白：cancellable 为 true，说明可以取消
```

**第4步：取消任务**
```bash
# 旁白：现在我们发送取消请求
curl -X POST "http://localhost:8000/api/documents/cancel/abc-123-def"

# 响应
{
  "success": true,
  "message": "取消请求已发送，任务将在下一个检查点停止",
  "document_id": "abc-123-def",
  "status": "cancelling"
}

# 旁白：取消请求已发送，任务将在下一个检查点停止
```

**第5步：等待并验证**
```bash
# 旁白：等待几秒钟，让任务完全停止
sleep 3

# 再次查询状态
curl "http://localhost:8000/api/documents/status/abc-123-def"

# 响应
{
  "status": "cancelled",
  "progress": 100,
  "message": "Task cancelled by user",
  "cancelled_at": "2024-01-01T12:00:00"
}

# 旁白：状态已变为 cancelled，取消成功！
```

## 🎯 演示要点

### 强调的功能点
1. ✅ **易用性**：一键取消，操作简单
2. ✅ **实时性**：状态实时更新，进度可见
3. ✅ **可靠性**：自动清理资源，数据一致
4. ✅ **灵活性**：支持前端和API两种方式

### 演示技巧
- 使用较大的文件（> 10MB）以便有足够时间演示
- 强调"正在处理的文档"列表功能
- 展示取消前后的状态对比
- 验证文档列表中确实没有被取消的文档

### 常见问题准备
Q: 取消后文件还在吗？  
A: 不在，系统会自动清理所有相关文件。

Q: 会影响其他任务吗？  
A: 不会，每个任务独立处理。

Q: 取消需要多久？  
A: 通常几秒钟内完成。
